I. Memory Layout Illusions

Endgame Trap

1. The Union Zero-Byte Trick (2014-15)

What is the output of the following program?

#include <stdio.h>
union som_union { 
    int num1; 
    char arr[4]; 
    int num2; 
}; 

int main() { 
    union som_union t; 
    t.num1 = 0; 
    t.arr[1] = 'A'; 
    printf("%s", t.arr); 
    return 0; 
}

Answer: Nothing is printed

The Trick: A union shares the exact same memory space for all its members.

  • t.num1 = 0; fills the 4 bytes of the union with zeros: [00][00][00][00]
  • t.arr[1] = 'A'; modifies the second byte (ASCII 65): [00][41][00][00]
  • printf("%s", t.arr); attempts to print a string starting at t.arr[0].
  • Because t.arr[0] is 00 (which is the Null Terminator '\0'), printf immediately stops reading. It prints absolutely nothing.
Conceptual

2. The Infinite Struct (2014-15)

struct folder { 
    int id; 
    struct folder subfolder; 
}; 

int main() { 
    struct folder f; 
    f.id = 10; 
    printf("%d ", f.id + 20); 
    return 0; 
}

Answer: Compile-time error

The Trick: A struct cannot contain an instance of itself by value.

If a folder contains a folder, which contains a folder, which contains a folder... the compiler cannot calculate the size of struct folder (it requires infinite memory). To create self-referential structures (like Linked Lists or Trees), you must use pointers (e.g., struct folder *subfolder;).

II. Operator Precedence Brutality

Sneaky

3. Assignment vs Comma Operator (2019-20)

int main() {
    int x; 
    float y; 
    x = 3, 17, 49; 
    y = (3.5, 7.1, 4.5); 
    printf("%f", x + y);
    return 0;
}

Answer: 7.500000

The Trick: The assignment operator = has higher precedence than the comma operator ,.

  • x = 3, 17, 49; is evaluated as (x = 3), 17, 49;. `x` becomes 3. The 17 and 49 are evaluated and discarded.
  • y = (3.5, 7.1, 4.5); uses parentheses to force the comma operator to evaluate first. The comma operator returns the right-most value. So y = 4.5.
  • `3 + 4.5 = 7.5`. As a float, it prints `7.500000`.
Pointers

4. Pre/Post-Increment Pointer Fusion (2024-25)

int main() {
    int a[5] = {11, 22, 33, 44, 55};
    int *p = a;
    printf("%d ", ++*p++);
    printf("%d", *p);
    return 0;
}

Answer: 12 22

The Trick: Deconstructing ++*p++.

  1. Postfix ++ has higher precedence than dereference *. So it evaluates as ++(*(p++)).
  2. p++ returns the current pointer address (pointing to `a[0]` which is 11), but queues `p` to advance to `a[1]`.
  3. The dereference * retrieves the value at `a[0]` (11).
  4. The prefix ++ increments that value. `a[0]` becomes 12. Prints 12.
  5. Next line: `p` has now advanced to `a[1]`. Dereferencing it prints 22.
Bitwise

5. Bitwise NOT on Negative Char (2019-20)

int main() {
    static int var; 
    char c = -2; 
    printf("%d", --var + ~c);
    return 0;
}

Answer: 0

The Trick: Manual 2's Complement bit manipulation.

  • `var` is static, so it defaults to `0`. `--var` makes it -1.
  • `c = -2`. In 8-bit binary (2's complement), positive 2 is 0000 0010. Invert and add 1 gives -2: 1111 1110.
  • The Bitwise NOT ~c flips all bits of `1111 1110`, resulting in 0000 0001. This is decimal 1.
  • `-1 + 1` = 0.

III. Deceptive Loop Mechanics

Master Trap

6. Scanf Loop Condition (2014-15)

How many times will the scanf() statement execute if the user continuously enters integers?

int main() {
    int t, i; 
    for (t = 2; scanf("%d", &i) - t; printf("%d\n", i)) {
        printf("%d--", t--); 
    }
    return 0;
}

Answer: 2 times

The Trick: scanf returns the number of successfully read items (which is `1`).

  1. Iteration 1 (Condition): `scanf` reads an integer and returns `1`. The condition is 1 - t. Since `t=2`, this is `1 - 2 = -1`. In C, any non-zero value is True. The loop enters.
  2. Iteration 1 (Body): Prints 2-- and decrements `t` to `1`.
  3. Iteration 1 (Update): Prints the value of `i` that was scanned.
  4. Iteration 2 (Condition): `scanf` executes a second time, returning `1`. The condition is 1 - t. Since `t` is now `1`, this evaluates to `1 - 1 = 0`. Zero is False. The loop breaks.

The `scanf` statement was evaluated exactly twice.

Pattern

7. The Standard Library Masquerade (2014-15)

Which standard C library function does this mimic?

void mysteryFunction(char *x, char *y) { 
    char *marker, *marker2; 
    for(marker = x, marker2 = y; *marker2; ++marker2, ++marker) {
        *marker = *marker2; 
    }
    *marker = *marker2; 
}

Answer: strcpy()

The Breakdown:
1. It initializes two pointers, one to string `x` (destination) and one to string `y` (source).
2. The loop condition *marker2 checks if the current character in source `y` is not the Null terminator `\0`.
3. In the loop, it copies the character: *marker = *marker2.
4. It increments both pointers.
5. When the loop ends (hits the Null terminator), the final statement *marker = *marker2 copies the actual `\0` byte over to the destination string, properly terminating it.
