1. Basics & Compilation Process

Memory Hierarchy (Fastest to Slowest)

  1. Registers (Inbuilt on processor, volatile)
  2. Cache (Inbuilt on processor, volatile)
  3. Primary Memory (RAM is volatile, ROM is non-volatile)
  4. Secondary Memory (HDD, SSD, Tape, CD - non-volatile)

The Compilation Process (4 Phases)

You can generate intermediate files using gcc -save-temps file.c

  1. Preprocessor (.i): Text manipulation. Removes comments, processes directives like #include and #define.
  2. Compiler (.s): Checks syntax, creates Assembly Code.
  3. Assembler (.o): Converts assembly language into Machine/Object code. Cannot be run yet.
  4. Linker (.exe or .out): Resolves external references (e.g., printf). Combines multiple object files into the final executable.
Crucial Exam Concept: Types of Errors
  • Syntax Error: Missing semicolon, open parenthesis, undeclared variable. Detected by Compiler.
  • Semantic Error: Valid syntax but conceptually meaningless. E.g., a * b = c; (error: lvalue required as left operand of assignment). Detected by Compiler.
  • Linker Error: Multiple definitions of a function or missing library object code when combining files.
  • Run-time Error: Divide by zero, Array index out of bounds, memory leaks, infinite loops missing break statements. Usually results in a crash or core dump.
  • Logical Error: Runs perfectly, but wrong output. E.g., using = instead of == in an if statement.

3. Deep Dive: Number Systems & IEEE 754 Floating Point

Negative Binary Numbers

The Most Significant Bit (MSB) acts as the sign bit (0 = positive, 1 = negative).

  • Signed-Magnitude: 1st bit is sign, next 7 bits are magnitude. (+0 and -0 exist). Range: -127 to +127.
  • 1's Complement: Invert all bits. (+0 and -0 exist). Range: -127 to +127.
  • 2's Complement: Invert all bits and add 1. Range: -128 to +127. This is the standard used in modern computing.
Rule of Thumb: For any positive number, its representation in Signed-Magnitude, 1's Complement, and 2's Complement is EXACTLY the same! It is just the binary representation padded with zeros to the left.

Subtraction using 2's Complement

To compute A - B: Take the 2's complement of B, and add it to A. Discard any carry beyond the MSB.

Example: 7 - 4 (using 8 bits)
7 = 00000111
4 = 00000100 -> 1's comp = 11111011 -> 2's comp = 11111100
Add:
  00000111 (7)
+ 11111100 (-4)
----------------
 100000011 -> Discard the 9th bit (1).
Result: 00000011 -> 3 in decimal.

IEEE 754 Floating Point Representation (32-bit Single Precision)

Formula: (-1)s × 1.m × 2(e - 127)

  • Sign bit (s) [1 bit]: Bit 31. 0 for positive, 1 for negative.
  • Exponent (e) [8 bits]: Bits 30 to 23. Uses Excess-127. You must ADD 127 to the actual binary exponent and store that value.
  • Mantissa/Significand (m) [23 bits]: Bits 22 to 0. The fractional part AFTER the implied 1..

Detailed Exam Example 1: Convert 10.6875 to IEEE 754 32-bit

  1. Convert integer part (10): 10 / 2 = 5 (R0), 5 / 2 = 2 (R1), 2 / 2 = 1 (R0), 1 / 2 = 0 (R1) → 1010
  2. Convert fractional part (0.6875):
    • 0.6875 × 2 = 1.375
    • 0.375 × 2 = 0.75
    • 0.75 × 2 = 1.5
    • 0.5 × 2 = 1.0 (Stops here)
    → .1011
  3. Combine: 1010.1011
  4. Normalize: Move decimal 3 places to the left: 1.0101011 × 23
  5. Calculate Exponent (e): Actual exponent is 3. Add 127. 3 + 127 = 130. Binary of 130 is 10000010.
  6. Extract Mantissa: Take the part after the decimal: 0101011. Pad with zeros to make it 23 bits: 01010110000000000000000.
  7. Sign bit: Positive number, so 0.
  8. Final Result: 0 10000010 01010110000000000000000

Detailed Exam Example 2: Convert 12.375 to IEEE 754 32-bit

  • 12 = 1100
  • 0.375 = .011
  • Combined = 1100.011
  • Normalized = 1.100011 × 23
  • Exponent = 130 → 10000010
  • Mantissa = 10001100000000000000000
  • Final Result: 0 10000010 10001100000000000000000

Detailed Exam Example 3: IEEE 754 to Decimal

Given: 1 10110110 01100000000000000000000

  • Sign: 1 → Negative number.
  • Exponent: 10110110 in decimal is 128 + 32 + 16 + 4 + 2 = 182. Actual exponent = 182 - 127 = 55.
  • Mantissa: 011... implies 1.011 in binary.
  • Calculation: -1 × 1.011 × 255
  • 1.011 in binary is 1 + 0/2 + 1/4 + 1/8 = 1.375.
  • Final Result: -1.375 × 255
Exception/Edge Case: Infinite Repeating Binary
What if you convert -10.7 to IEEE 754?
The fractional part 0.7 yields an infinite repeating pattern: .10110011001100...
Rule: You store only the bits that fit the 23-bit mantissa. The rest are truncated. This leads to approximation/precision loss (e.g., it might read back as -10.699999).

4. Data Types, Variables & Operators

Type Hierarchy & Conversions

Implicit conversion promotes to the largest type: int → unsigned int → long → float → double → long double.
char and short are always converted to int before arithmetic.

float A = 100/3;     // Output: 33.000000 (Integer division first)
float B = 100/3.0;   // Output: 33.333333 (Float division)
int C = 10; C = C + 'B'; // 'B' (66) converted to int. C = 76.

Operator Precedence and Tricks

  • Unary operators (++, --, !, *, &) have Right-to-Left associativity.
  • Arithmetic *, /, % > +, -
  • Bitwise Operators: & (AND), | (OR), ^ (XOR), ~ (NOT).
Short-Circuiting Rules (Crucial for output tracing):
  • AND (&&): Evaluates Left to Right. If the left is 0 (False), the right side is IGNORED.
  • OR (||): Evaluates Left to Right. If the left is non-zero (True), the right side is IGNORED.
int a = 0, b = 3;
int I = a++ && ++b; 
// a++ returns 0 (then increments a to 1). Left is False.
// ++b is SKIPPED entirely.
printf("%d %d %d", a, b, I);
1 3 0
int a = 1, b = 1;
int c = a || --b; // a is 1 (True). --b is SKIPPED. c = 1.
int d = a-- && --b; // a-- returns 1 (True). --b executes (b becomes 0, returning 0). d = 0.
printf("a=%d, b=%d, c=%d, d=%d", a, b, c, d);
a=0, b=0, c=1, d=0

5. Control Flow: Branching & Looping

Loops: while, do...while, for

Entry-controlledExit-controlled (runs at least once)Entry-controlled, best for known iterations
Loop Exceptions & Edge Cases:
  • while(num > 0); - The semicolon terminates the loop. If num>0, it's an infinite loop doing nothing.
  • for(int i=1, j=10; i<10 && j>0; i++, j--) - Multiple variables can be managed.
  • break; - Immediately suspends the current/innermost loop.

6. Functions, Scope & Storage Classes

Memory Layout during Execution

  • Text Segment: Compiled machine code.
  • Data Segment: Global and Static variables. Initialized here.
  • Heap Segment: Dynamic allocation (malloc). Grows upwards.
  • Stack Segment: Local (auto) variables, function frames. Grows downwards. Destroyed on return.

Static & Register Exceptions:
  • Static variables are initialized ONLY ONCE. If static int y = 0; is inside a loop, it will NOT reset to 0 on the second iteration.
  • You CANNOT access the memory address of a register variable using &. Therefore, scanf("%d", ®_var); will cause a compilation error.

    static int i = 0;

7. Arrays & Strings

Arrays

  • No Bounds Checking: C does not stop you from accessing arr[10] in an array of size 5. It will overwrite other memory, causing bugs or "Stack Smashing" errors.
  • Assignment: You cannot do arr1 = arr2;. You must copy element-by-element using a loop.
  • Arrays are always passed to functions by reference (pointer to the first element).

Strings (<string.h>)

Strings are character arrays terminated by '\0'. char str[4] = "BITS"; is an ERROR because there is no space for the null terminator.

  • scanf("%s", str) reads until a space.
  • scanf("%[^\n]s", str) reads a full line until Enter.
  • gets(str) reads a full line and auto-appends \0.
// String concatenation implementation logic manually:
int i = 0, j = 0;
while(s1[i++] != '\0'); i--; // Go to end of s1
while(s2[j] != '\0') {
    s1[i++] = s2[j++];
}
s1[i] = '\0';

Key String Functions:

  • strcpy(s1, s2): Copies s2 into s1.
  • strcmp(s1, s2): Returns 0 if identical. <0 if s1 is alphabetically before s2. >0 if s1 after s2.
  • strstr(s1, s2): Returns pointer to first occurrence of s2 in s1, or NULL.
  • atoi(str): Converts string to integer (needs <stdlib.h>).
  • atof(str): Converts string to double.

8. Pointers & Pointer Arithmetic

Pointers store memory addresses. int *p = &a;

  • Null Pointer: int *p = NULL; (Always do this to avoid dangling pointer issues).
  • Void Pointer: void *p; Generic pointer. Must be typecast to dereference: *(int *)p.

Pointer Arithmetic

Adding K to a pointer moves the address by K * sizeof(type).

int a[5] = {5, 10, 15, 20, 25};
int *ptr = a;  // Assume ptr address is 1000, int is 4 bytes
ptr = ptr + 2; // Address becomes 1008. Points to 15.
Tracing Pointer Expressions (Exam Favorites):
  • *p++ or *(p++) : Evaluates to the value at p, then the pointer p moves to the next address.
  • *++p or *(++p) : Pointer p moves to the next address, then evaluates the value at that new address.
  • ++*p or ++(*p) : Increments the value stored at address p. The address does not change.
  • (*p)++ : Evaluates the value at p, then increments the value in memory. Address does not change.
int main() {
    int v = 3, *pv;
    pv = &v;
    (*pv)++;
    printf("v=%d, *pv=%d\n", v, *pv);
}
v=4, *pv=4

Array-Pointer Equivalence

arr[i] is exactly equivalent to *(arr + i).

For 2D arrays: arr2D[i][j] is exactly equivalent to *(*(arr2D + i) + j).

9. Structures, Unions & Enums

Structures

A custom data type grouping logically related variables.

struct student {
    int ID;
    char Name[20];
}; // Don't forget the semicolon!
struct student s1;
Structure Traps:
  • Padding: sizeof(struct) is often larger than the sum of its parts. E.g., a struct with a short (2 bytes), int (4 bytes), and char (1 byte) might take 12 bytes instead of 7 due to compiler alignment padding.
  • Assignment: s1 = s2; is perfectly legal. It copies all fields.
  • Comparison: if(s1 == s2) is ILLEGAL in C. You must compare field-by-field: if(s1.ID == s2.ID).

Pointers to Structures

struct student *ptr = &s1;
printf("%d", ptr->ID);      // Correct
printf("%d", (*ptr).ID);    // Correct
// printf("%d", *ptr.ID);   // WRONG! Dot operator has higher precedence than *.

Enumerators (enum)

Restricts variables to a specific set of integer values.

typedef enum {A1=1, A2=2, A3=5} Group_lbl;
Group_lbl myGroup = A3; // Internally treated as integer 5

11. Linked Lists

Uses Self-Referential Structures (contains a pointer to its own type).

struct node {
    int data;
    struct node *next;
};

Singly Linked List

One-way traversal. The next pointer of the last node is NULL.

// Insertion at Beginning
void insertFront(struct node **head, int val) {
    struct node *newNode = (struct node*) malloc(sizeof(struct node));
    newNode->data = val;
    newNode->next = *head;
    *head = newNode;
}

// Deletion at Beginning
void deleteFront(struct node **head) {
    if(*head == NULL) return;
    struct node *temp = *head;
    *head = (*head)->next;
    free(temp); // Crucial to prevent memory leaks!
}

Doubly Linked List

Nodes contain prev and next pointers. Enables two-way traversal.

struct dllnode {
    int data;
    struct dllnode *prev;
    struct dllnode *next;
};

Circular Linked List

The next of the last node points back to the head node instead of NULL.

Cycle Traps: In a circular linked list, iterating until curr != NULL will result in an infinite loop!

Detecting Cycles (Floyd's Algorithm)

Traverse the list using two pointers: slow_p (moves 1 node) and fast_p (moves 2 nodes). If they eventually point to the same node, a cycle exists. If either hits NULL, no cycle exists.

File Handling

File Modes

  • "a": Append. Creates new or adds to end.
  • "r+", "w+", "a+": Opens for BOTH read and write.

Functions: fgetc, fputc, fgets, fputs, fscanf, fprintf.
rewind(fp); moves the file position indicator back to the very beginning of the file.

Command Line Arguments

int main(int argc, char **argv) {
    // argc: argument count
    // argv: array of string pointers
}
  • argv[0] is ALWAYS the name of the executable (e.g., "./a.out").
  • The first user-provided argument is at argv[1].
  • If you pass numbers (e.g. ./calc 5 10), they are strings. You must convert them: int a = atoi(argv[1]);

13. Recursion

Recursion is a function calling itself. A valid recursive function MUST have a Base Case (a condition to return without making another recursive call) to prevent infinite loops (Stack Overflow).

// Factorial
long int factr(int n) {
    if(n == 1) return 1; // Base case
    return factr(n-1) * n; // Recursive call
}

// Power Function
double power(double val, unsigned pow) {
    if (pow == 0) return 1.0; // Base case
    return power(val, pow-1) * val;
}

How it works in memory

  • Every recursive call pushes a new frame onto the Stack Segment.
  • Local variables are distinct in every frame.
  • When a base case is hit, the function returns, the top frame is popped, and the caller resumes execution.

Pros:

  • Results in cleaner, simpler, and more readable code.
  • Essential for complex algorithms (MergeSort, QuickSort, Tree Traversals).

Cons / Limits:

  • No reduction in compiled code size.
  • Slower execution due to heavy overhead of function calls (saving registers, allocating frames).
  • High risk of Stack Overrun (Stack Overflow) if recursion depth is too high.

Tracing Fibonacci Time Complexity

int fib(int num) {
    if(num == 0) return 0;
    if(num == 1) return 1;
    return fib(num-1) + fib(num-2);
}

Number of times fib is called:
Input 0 → 1 call
Input 1 → 1 call
Input 2 → 3 calls
Input 3 → 5 calls
Input 4 → 9 calls
Input 5 → 15 calls...
Notice the exponential growth! This is highly inefficient compared to an iterative approach.
