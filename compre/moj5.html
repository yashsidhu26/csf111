1. The IEEE-754 Hex Trap (2021-22)

A system stores floating-point numbers in IEEE-754 Single Precision (32-bit). The hexadecimal content of registers A, B, and C are 0xC1400000, 0x42100000, and 0x41400000 respectively. What is the output?

if (C == A + B)
    printf("Condition 1 is true");
else if (A + C == 0 && B == 3 * C)
    printf("Condition 2 is true");
else
    printf("None of the conditions is true");

Answer: Condition 2 is true

The Decode: You must convert Hex to IEEE-754 Binary (1 sign bit, 8 exponent bits, 23 mantissa bits).

  • A (0xC1400000): `1` `10000010` `1000...0`
    Sign = 1 (Negative). Exp = 130 - 127 = 3. Mantissa = 1.1
    Value = -1.1 * 2³ = -1100 in binary = -12.0
  • B (0x42100000): `0` `10000100` `0010...0`
    Sign = 0 (Positive). Exp = 132 - 127 = 5. Mantissa = 1.001
    Value = +1.001 * 2⁵ = 100100 in binary = 36.0
  • C (0x41400000): `0` `10000010` `1000...0`
    Sign = 0 (Positive). Exp = 130 - 127 = 3. Mantissa = 1.1
    Value = +1.1 * 2³ = +1100 in binary = +12.0

Evaluating Logic:
`C == A + B` -> 12 == -12 + 36 -> 12 == 24 (False)
`A + C == 0` -> -12 + 12 == 0 (True) AND `B == 3 * C` -> 36 == 3 * 12 (True).

2. The 32 vs 64-Bit Pointer Trap (2020-21)

Assume this code is compiled and run on a modern 64-bit system.

#include <stdio.h>

void function(char **ptr) {
    char *ptr1;
    ptr1 = (ptr += sizeof(ptr))[-4];
    printf("%s\n", ptr1);
}

int main() { 
    char *arr[] = { "ant", "bat", "cat", "dog", "bee", "fly" };
    function(arr); 
    return 0; 
}

Answer: bee

The Trick: The result of this code depends entirely on the hardware architecture!

  • ptr is a double pointer (char **).
  • On a 64-bit system, the size of ANY pointer is 8 bytes.
  • ptr += sizeof(ptr) translates to ptr += 8. It moves the pointer forward by 8 indices.
  • Then it accesses [-4]. This translates to *(ptr - 4).
  • Net movement: ptr + 8 - 4 = ptr + 4.
  • arr[4] corresponds to the string "bee".

Note: If this were a 32-bit system, sizeof(ptr) would be 4, net movement would be 4 - 4 = 0, printing "ant"!

3. Precedence Reality Distortion (2014-15)

The expression 5 - 2 - 3 * 5 - 2 will mathematically evaluate to 18 IF AND ONLY IF:

A) - is left associative and * has precedence over -

B) - is right associative and * has precedence over -

C) - is right associative and - has precedence over *

D) - is left associative and - has precedence over *

Answer: Option C

The Proof: Standard C (Option A) evaluates this as (5 - 2) - (3 * 5) - 2 = -14.
To get 18, we have to build a theoretical Abstract Syntax Tree based on the given rules.

Let's test Option C: Minus is computed first, and it computes right-to-left.

  • Expression: 5 - 2 - 3 * 5 - 2
  • Because - has higher priority than *, we must resolve all subtractions before the multiplication.
  • Because it is Right Associative, we resolve the right-most subtraction first: (5 - 2) = 3.
  • The next right-most subtraction is 2 - 3 = -1.
  • The final subtraction is 5 - (-1) = 6.
  • Now that all subtractions are resolved, we perform the multiplication: 6 * 3 = 18!

4. Partial Array Initialization (2023-24)

int main() {
    int a[10] = {2, 2, 4, 4, 4};
    int *ptr = a;
    printf("%u %u", ptr[5], *ptr + 6);
    return 0;
}

Answer: 0 8

The Trick: Two fundamental C rules interacting.

  1. Partial Initialization: If you initialize any elements of an array, the compiler automatically fills all remaining uninitialized elements with 0. Therefore, `a[5]` to `a[9]` are all `0`. So ptr[5] prints 0.
  2. Pointer Arithmetic Precedence: *ptr + 6 means "Dereference `ptr` first, then add 6 to the value." `ptr` points to `a[0]` (which is 2). `2 + 6 = 8`.

5. 3D Pointer Inception (2020-21)

#include <stdio.h>

int f(int a, int *pb, int **ppc) {
    static int b, c;
    **ppc += 1;
    c = **ppc;
    *pb += 2;
    c = *pb;
    a += 3;
    return a + b + c;
}

int main() {
    int c = 4, b = 0, a = 0;
    b = (int)&c; 
    a = (int)&b;
    printf("%d", f(c, (int*)b, (int**)a));
    return 0;
}

Answer: 14

The Trick: Tracking local variables vs pointers to `main`'s variables, alongside local static shadowing.

  • f(c, &c, &b) is called. Local `a`=4, `pb` points to `main`'s `c`, `ppc` points to `main`'s `b`.
  • **ppc += 1; increments the value pointed to by `b` (which is `main`'s `c`). So `main`'s `c` becomes 5.
  • c = **ppc; -> The local static `c` is assigned 5.
  • *pb += 2; -> `pb` points to `main`'s `c`. It increments from 5 to 7.
  • c = *pb; -> Local static `c` updates to 7.
  • a += 3; -> Local `a` (which was 4) becomes 7.
  • Returns a + b + c (using local statics). `a`=7, `b`=0 (uninitialized static defaults to 0), `c`=7. `7 + 0 + 7 = 14`.

6. The Static Pointer Escape (2024-25)

#include <stdio.h>

void update(int **p) {
    static int val = 20;
    *p = &val;
}

int main() {
    int x = 10; 
    int *ptr = &x;
    
    update(&ptr);
    printf("%d\n", *ptr);
    return 0;
}

Answer: 20

The Trick: A classic beginner rule is "Never return a pointer to a local variable because it gets destroyed when the function exits."

However, because val is declared as static, it is stored in the Data Segment, not the Stack. Its lifetime is the entire duration of the program.

By passing &ptr (a double pointer), the update() function successfully repoints `main`'s `ptr` to the static `val`. It safely prints 20.
