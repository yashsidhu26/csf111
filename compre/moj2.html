I. Memory & Compiler Quirks

Lethal Trap

1. The sizeof() Illusion (2019-20)

#include <stdio.h>
int main() {
    static int var; 
    int new_val = sizeof(var++) + ++var; 
    printf("var = %d", var);
    return 0;
}

Answer: var = 1

The Trick: The sizeof operator evaluates the type of its operand at compile-time. It DOES NOT execute the code inside it at runtime.

Therefore, var++ inside the `sizeof` is completely ignored during execution! `var` starts at 0 (since it's static), the `sizeof` evaluates to 4 (bytes), and then ++var executes, making `var` = 1.

Memory

2. VLA vs Function Decay (2014-15)

Assume this runs on a 64-bit machine where pointers are 8 bytes and ints are 4 bytes.

void printSize(int arr[]) { 
    printf("%lu ", sizeof(arr)); 
}

int main() { 
    int SIZE = 10; 
    int arr[SIZE]; 
    printf("%lu ", sizeof(arr)); 
    printSize(arr); 
    return 0; 
}

Answer: 40 8

The Trick: Inside `main`, `arr` is an actual array. sizeof(arr) correctly identifies it as 10 integers (10 * 4 = 40).

However, when passed to a function, arrays decay into pointers. Inside `printSize`, `arr` is strictly treated as an int*. Therefore, `sizeof(arr)` returns the size of a pointer, which is 8 bytes on a 64-bit system.

Scope

3. The Static Constraint (2019-20)

int main() {
    int i = 10; 
    static int x = i;
    if(x == i) 
        printf("One"); 
    else 
        printf("Two");
    return 0;
}

Answer: Compile-time Error

The Trick: In C, variables declared as static are allocated in the data segment before the program even starts running. Therefore, they must be initialized with a constant expression.

You cannot initialize a static variable with a local, runtime variable like `i`. The compiler will flag this as: "initializer element is not constant".

II. Pointer Arithmetic & ASCII Math

Brilliant

4. ASCII Pointer Shifts (2016-17)

int main (void) { 
    char c[] = "KATEWINCE"; 
    char *p = c; 
    printf("%s", p + p[3] - p[1]) ; 
    return (0); 
}

Answer: WINCE

The Trick: You can perform pointer arithmetic using the underlying integer (ASCII) values of characters!

1. `p[3]` is the character 'E' (ASCII 69).
2. `p[1]` is the character 'A' (ASCII 65).
3. The expression evaluates to: p + 69 - 65 -> p + 4.
4. Adding 4 to pointer `p` shifts it 4 characters forward, making it point to index 4 (the letter 'W'). Printing it as a string yields "WINCE".

Subtle

5. The Unmoved Pointer Loop (2014-15)

int main() {
    int num[] = {0, 4, 2, 6, 4, 8};
    int *n = num + 3; 
    int j = 1, val = 0; 
    
    while(j++ <= 3) { 
        val += (*(n+1) > 4 ? 1 : -1); 
    } 
    printf("%d", val);
    return 0;
}

Answer: -3

The Trick: Look closely at the loop body. The pointer `n` is never incremented or updated!

`n` points to `num[3]` (which is 6). Inside the loop, *(n+1) looks at `num[4]` (which is 4).
It asks: Is 4 > 4? No, so the ternary returns `-1`.
The loop runs exactly 3 times (for j=1, 2, 3), and every single time it evaluates the exact same static pointer expression, adding -1 to `val`.
Final result: -3.

III. Bugs, Files & Enums

Trap

6. Missing Null Terminator (2022-23)

#include <string.h>
#include <stdio.h>
int main() { 
    char a[15]; 
    char *s = "Hello BITS"; 
    int len = strlen(s); 
    
    for(int i=0; i<len; i++) 
        a[i] = s[len-1-i]; 
        
    printf("%s", a); 
    return 0; 
}

Answer: STIB olleH[Garbage Values]

The Trick: The code successfully reverses the string character by character into array `a`. However, it forgets to append the null-terminator '\0' at `a[len]`.

When passed to `printf("%s")`, it prints the reversed characters but continues reading adjacent memory until it randomly hits a null byte, resulting in trailing garbage characters.

File I/O

7. Uninitialized do-while (2024-25)

int main() {
    FILE *pt = fopen("test.txt", "r"); 
    char ch; 
    do { 
        printf("%c", ch); 
    } while((ch = fgetc(pt)) != EOF);
    
    fclose(pt);
    return 0;
}

Answer: Prints a garbage character before the file content.

The Trick: A do-while loop executes the body *before* checking the condition.

In the very first iteration, `printf` prints the variable `ch`. However, `ch` has just been declared and hasn't been assigned anything from the file yet. It contains memory garbage. Only after the first print does it execute `fgetc` inside the `while` condition.

Scoping

8. Enum Overrides (2016-17)

int main (void) { 
    enum tak {_2 , _3 , _4, _1}; 
    enum cak {_5, _6, _7 }; 
    
    int a = _3 * 6 / _7 % (_1 - 1); 
    printf ("a=%d\n", a); 
    return 0;
}

Answer: a=1

The Trick: Enums start assigning values at 0 by default, and this resets for each new enum declaration. Also, identifiers starting with underscores are perfectly legal.

- From enum tak: `_2`=0, `_3`=1, `_4`=2, `_1`=3.
- From enum cak: `_5`=0, `_6`=1, `_7`=2.

Substitute the math: 1 * 6 / 2 % (3 - 1)
Following precedence (Left to Right for *, /, %):
`6 / 2 % 2` -> `3 % 2` = 1.
