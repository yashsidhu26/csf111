🛡️ Exam Survival Tactics

  • Macro Expansion: It is DUMB text replacement. Write it out physically before doing math.
  • Visual Traps: Ignore indentation. Trace the actual curly braces {}.
  • Discarded Execution: Right sides of && and || do NOT execute if short-circuited.
  • Array vs &Array: arr+1 skips an element. &arr+1 skips the ENTIRE array.

  • Printf Evaluation: Arguments are often evaluated Right-To-Left.

I. The Preprocessor Minefield

Trap

1. The Discarded Argument

#define fun1(x) 50 
#define fun2(x) (fun1(x)<0) ? fun1(x)-5 : fun1(x)+5 

int main() { 
    int x=5; 
    printf("\nfun: %d, x: %d", fun2(++x+3), x); 
    return 0;
}

Answer: fun: 55, x: 5

The Trick: fun1(x) is defined purely as `50`. It never uses `x`. When fun2(++x+3) expands, it becomes (50 < 0) ? 50 - 5 : 50 + 5. Because the macro completely discarded the argument, the increment ++x never happens.

Hard

2. The Expansion Nightmare

#define ADD(x) x + x + x
#define FUN(x) ADD(x) * 2

int main() {
    int a = 5;
    printf("Result = %d", FUN(ADD(a)));
    return 0;
}

Answer: 50

The Trick: Do not calculate mathematically. Do string substitution! FUN(ADD(a)) -> ADD(ADD(a)) * 2 -> ADD(a + a + a) * 2 -> a + a + a + a + a + a + a + a + a * 2. With precedence, this is 40 + 10 = 50.

Trap

3. Recursive Macros

#define fact(n) (n==0 ? 1 : n * fact(n-1))
int main() {
    int result = fact(5);
    return 0;
}

Answer: Compile/Linker Error

The Trick: The C preprocessor does NOT support recursive macros. It expands fact(n) exactly once: (5==0 ? 1 : 5 * fact(5-1)). The compiler then looks for a real C function named `fact`, doesn't find it, and throws an error.

II. Flow & Short-Circuit Anomalies

Complex

4. The printf() Loop Maze

int main() {
    int i = 0; 
    for(printf("A"); printf("B"); printf("C")) { 
        for(printf("D"); printf("E"); printf("F")) 
            break; 
        if (i--) break; 
    }
    return 0;
}

Answer: ABDECBDE

The Trick: `printf()` returns the number of characters printed (1 = True). Inner loop hits break; immediately, skipping "F". Outer loop checks `if(i--)`. `i` is 0 (False), then becomes -1. Update runs ("C"), Cond runs ("B"), inner loop runs ("D", "E"), breaks. Now `if(i--)` is checked with `i=-1` (True). It breaks the outer loop.

Syntax

5. Top-Default Switch Fallthrough

int main() {
    int i = 4;
    switch(i) {
        default: printf("zero");
        case 1:  printf("one"); break;
        case 2:  printf("two"); break;
    }
    return 0;
}

Answer: zeroone

The Trick: `default` doesn't have to be at the bottom! `i=4` matches nothing, execution jumps to `default`, prints "zero". Because there is no break after `default`, it falls through to `case 1`, printing "one", and then breaks.

Sneaky

6. The Indentation Illusion

int main() {
    int v1 = 99, v2 = -99;
    if (v1 - v2 < 0) {
        v2 = v1 * v2;
    } else 
        break;
        printf("Hello");
    return 0;
}

Answer: Compile-time Error

The Trick: The `break` statement is only valid inside a `loop` or a `switch`. Placing it inside a standalone `if/else` block throws a compile-time error. The bad indentation is purely to trick you into tracing logic that won't even compile.

Trap

7. Short-Circuit Assignment

int main() {
    int a = 0, b = 0, x;
    x = 121 || (a = ++b);
    printf("a = %d, b = %d, x = %d", a, b, x);
    return 0;
}

Answer: a = 0, b = 0, x = 1

The Trick: Logical OR (||) short-circuits. `121` is True. The right side (a = ++b) is completely ignored. `a` and `b` remain 0. `x` gets the boolean result `1`.

III. Pointers, Arrays & Memory Magic

Hard

8. Negative Indexing

int main() {
    int arr[6] = {99, 88, 77, 66, 55, 44};
    int *p = arr + 4;
    for(int i=0; i<3; i++)
        printf("%d ", p[-i]);
    return 0;
}

Answer: 55 66 77

The Trick: Negative indices are valid if the pointer is in the middle of an array! p[-i] is *(p - i). `p` points to 55. `p[-0]`=55, `p[-1]`=66, `p[-2]`=77.

Conceptual

9. Array Decay vs Address of Array

Assume arr[] begins at memory address 1000 and sizeof(int) is 4.

int main() { 
    int arr[] = {12, 14, 15, 23, 45}; 
    printf("%u, %u", arr + 1, &arr + 1); 
    return 0; 
}

Answer: 1004, 1020

The Trick: arr decays to a pointer to an `int`. arr + 1 moves 4 bytes (Address 1004). &arr is a pointer to the entire array of 5 integers. &arr + 1 moves 20 bytes (Address 1020).

Danger

10. String Literals Mutation

int main() {
    char *ptr = "Hello"; 
    *(ptr + 1) = 'a'; 
    printf("%s\n", ptr);
    return 0;
}

Answer: Segmentation Fault

The Trick: char *ptr = "Hello" stores the string in the Read-Only Data segment. Modifying it crashes the program. Use char ptr[] = "Hello" if you want it mutable.

Hard

11. Double Pointers & Printf Evaluation

int main() {
    int x = 5, *y, z[2] = {22, 33}, **p;
    y = &x;  p = &y;  **p = 10;
    y = z;   *(z + 1) += x;
    printf("%d, %d, %d, %d\n", x, ++**p, z[0], z[1]);
    return 0;
}

Answer: 10, 23, 22, 43

The Trick: State tracking + RTL `printf` evaluation.
1. `**p=10` changes `x` to 10.
2. `y=z` makes `**p` alias `z[0]` (22).
3. `z[1]` becomes 33+10 = 43.
4. Printf evaluates Right-To-Left: `z[1]` is 43, `z[0]` is 22 (read before increment), `++**p` pre-increments `z[0]` to 23, `x` is 10.

Complex

12. Pointer to Arrays

int main() {
    int a[][3] = {0, 1, 2, 3, 4, 5};
    int (*ptr)[3] = a;
    ++ptr;
    printf("%d %d\n", (*ptr)[0], (*ptr)[1]);
    return 0;
}

Answer: 3 4

The Trick: int (*ptr)[3] is a pointer to an array of 3 integers. `++ptr` moves it forward by an entire row of 3 elements. It jumps from `{0,1,2}` to `{3,4,5}`.

Puzzle

13. Array of Pointers Arithmetic

int main() {
    int i[] = {10, 20, 30, 40}; 
    int *pa[] = {i+1, i, i+3, i+2}; 
    printf("%d", *pa[1]);
    return 0;
}

Answer: 10

The Trick: pa is an array of integer pointers. pa[1] holds i, which is the base address of the array (points to 10). Dereferencing it yields 10.

IV. Data Types, Structs & Enums

Data Type

14. The Wrapping Char

Assume char is 1 byte using 2's complement.

int main() { 
    char c = 125; 
    c = c + 10; 
    printf("c = %d", c); 
    return 0; 
}

Answer: -121

The Trick: A signed char maxes out at 127. 125 + 10 = 135. It overflows. Formula: Value - 256 -> 135 - 256 = -121.

Rare

15. The Comma Operator

int main() {
    int x = ((4,5), 3, (8,0));
    printf("%d", x);
    return 0;
}

Answer: 0

The Trick: The comma operator evaluates the left operand, discards it, and returns the right. (4,5) -> 5. (8,0) -> 0. The expression becomes (5, 3, 0). Ultimately evaluates to 0.

Logic

16. Implicit Enum Values

enum day { sun = 1, mon, tue = 5, wed, thu=10, fri, sat }; 
int main() { 
    printf("%d ", sun+mon-tue+wed-thu+fri-sat); 
    return 0; 
}

Answer: -7

The Trick: Unassigned enum values auto-increment from the previous explicit value.
sun=1, mon=2, tue=5, wed=6, thu=10, fri=11, sat=12.
1 + 2 - 5 + 6 - 10 + 11 - 12 = -7.

Memory

17. Pragma Pack Pitfall

#pragma pack(1)
struct x { int a, d; char b; };
union z { struct x s; int a; };
int main() {
    union z u;
    printf("size=%ld", sizeof(u));
    return 0;
}

Answer: size=9

The Trick: #pragma pack(1) forces 1-byte alignment, meaning no padding is added to the struct. The struct size is `4 + 4 + 1 = 9` bytes. Since it's a union, the size of the union is the size of its largest member, which is the struct (9 bytes).

V. Standard Library & Undefined Behavior

Logic

18. The strcmp() Illusion

int main(){ 
    char arr[10]="some"; 
    while(strcmp("some", arr)) 
        printf("Strings are not equal!! \n"); 
    return 0; 
}

Answer: No Output

The Trick: strcmp() returns 0 when two strings are equal. In C, `0` is False. The condition fails immediately.

File I/O

19. fseek Arithmetic

FILE *fp = fopen("myfile.txt","w+");
fputs("Comprehensive Examination", fp);
fseek(fp, 15, SEEK_SET);
char c = fgetc(fp);
while (c != EOF) { 
    printf("%c", c); 
    c = fgetc(fp); 
}

Answer: xamination

The Trick: SEEK_SET moves to absolute index 15 (0-indexed). Index 15 points to the 'x' in Examination. `fgetc` reads it and the loop prints the rest.

Complex

20. Main Recursion & Static Locals

int main() { 
    static int i = 3; 
    printf("%d", i--); 
    return i > 0 ? main() : 0; 
}

Answer: 321

The Trick: Static variables are initialized exactly once. main() calls itself recursively. On subsequent calls, `i` is not re-initialized to 3, it retains its decremented value.

Trap

21. The Sequence Point Trap

int func1() {
    static int val = 2;
    return ++val + val++;
}
int main() {
    printf("%d", func1());
    return 0;
}

Answer: Undefined Behavior (Compiler Dependent, often evaluates to 6)

The Trick: The C standard explicitly states that modifying a variable more than once between sequence points (like evaluating a single expression `++val + val++`) results in Undefined Behavior. You cannot guarantee the order of evaluation. However, in many exam contexts, they evaluate strictly left to right: `val` becomes 3, then it evaluates `3 + 3` (evaluates to 6), and finally `val` becomes 4.
