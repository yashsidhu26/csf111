1. The Recursive Prime Cipher (2022-23)

Trace this function carefully. What values will it output?

#include <stdio.h>

int q(int a, int b) {
    if (a == b) 
        return 1;
    else if (a % b) 
        return q(a, b + 1);
    else 
        return 0;
}

int main() {
    printf("%d %d %d\n", q(35, 2), q(11, 2), q(23, 2));
    return 0;
}

Answer: 0 1 1

The Algorithm: This recursive function is actually a disguised Prime Number Checker.

  • a % b returns a truthy value if there is a remainder (i.e., `b` does not perfectly divide `a`). If it doesn't divide, it recursively checks the next number: q(a, b + 1).
  • If `a % b` is 0 (it divides perfectly), it hits the else block and returns `0` (Not Prime).
  • If `b` increments all the way up to `a` without ever dividing it perfectly, the base case a == b triggers, returning `1` (Is Prime).
  • q(35, 2) -> Divisible by 5. Returns 0.
  • q(11, 2) -> Prime. Returns 1.
  • q(23, 2) -> Prime. Returns 1.

2. The Binary Mirror (2022-23)

#include <stdio.h>

void find(int n) {
    if (n == 0) return;
    
    printf("%d", n % 2);
    find(n / 2);
    printf("%d", n % 2);
}

int main() {
    find(12);
    return 0;
}

Answer: 00111100

The Algorithm: This function computes the binary digits of `n`, but the dual `printf` statements create a mirrored output.

  1. n=12: prints `12%2` (0). Calls find(6).
  2. n=6: prints `6%2` (0). Calls find(3).
  3. n=3: prints `3%2` (1). Calls find(1).
  4. n=1: prints `1%2` (1). Calls find(0).
  5. n=0: hits base case. Returns.
  6. Stack unwinds...
  7. n=1: prints `1%2` (1).
  8. n=3: prints `3%2` (1).
  9. n=6: prints `6%2` (0).
  10. n=12: prints `12%2` (0).

The output is the digits appended together: 00111100.

3. The Dangling Array Trap (2021-22)

#include <stdio.h>

int* return_arr() {
    int arr[] = {1, 4, 5};
    return arr;
}

int main() {
    int *p = return_arr();
    printf("%d", *p);
    return 0;
}

Answer: Undefined Behavior (Dangling Pointer)

The Trick: The array arr[] is created locally on the stack inside the function `return_arr()`. When the function finishes executing, that stack frame is destroyed and handed back to the OS.

Returning `arr` passes back a memory address that the program no longer owns. While it might occasionally print `1` by pure luck, it is a classic Dangling Pointer and will often print garbage or crash.

4. The Matrix Diagonal Cipher (2014-15)

What does this matrix traversal logic actually output?

int main() {
    int mat[3][3] = { {1, 2, 3}, 
                      {4, 5, 6}, 
                      {7, 8, 9} };
    int q = mat[0][2]; 
    
    for (int i = 1; i < 3; i++) { 
        if (mat[i][2-i] < q) { 
            q = mat[i][2-i]; 
        } 
    }
    printf("%d", q);
    return 0;
}

Answer: 3

The Algorithm: The index formula mat[i][N - 1 - i] (where N is the matrix size) is the mathematical formula to traverse the Secondary Diagonal of a square matrix (top-right to bottom-left).

  • i=0 (initial): mat[0][2] -> 3
  • i=1: mat[1][1] -> 5. Is 5 < 3? No.
  • i=2: mat[2][0] -> 7. Is 7 < 3? No.

The variable `q` holds the minimum value found on the secondary diagonal, which remains 3.

5. Recursive String Reversal (2024-25)

#include <stdio.h>

void strFunc(char A[], int n) {
    char t;
    if (n <= 1) return;
    
    t = A[0]; 
    A[0] = A[n - 1]; 
    A[n - 1] = t;
    
    strFunc(&A[1], n - 2);
}

int main() {
    char A[] = "PDS 2005";
    strFunc(A, 8);
    printf("%s", A);
    return 0;
}

Answer: 5002 SDP

The Algorithm: This is a highly optimized recursive string reversal function.

It swaps the very first character A[0] with the very last character A[n-1]. Then, it calls itself, passing the address of the second character &A[1] as the new "base" of the array, and reduces the length window by 2 (since the outer two chars are already swapped). This cleanly reverses the string in place.
