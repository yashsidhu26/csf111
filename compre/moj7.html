I. Mathematical Execution

Int Division Trap

1. The Truncated Memory Counter (2022-23)

How many total integer elements are successfully allocated across the entire 2D array?

#include <stdlib.h>
#include <stdio.h>

int main() {
    int **arr2d = (int **)malloc(3 * sizeof(int *)); 
    int total_elements = 0;
    
    for (int i = 0; i < 3; i++) {
        int size = (i / 2 + i + 3);
        arr2d[i] = (int *)malloc(size * sizeof(int)); 
        total_elements += size;
    }
    
    printf("Total = %d", total_elements);
    return 0;
}

Answer: 13

The Trick: Integer division! In C, dividing an integer by an integer entirely truncates the decimal part. It does not round up.

  • i = 0: (0 / 2) + 0 + 3 = 0 + 0 + 3 = 3 elements.
  • i = 1: (1 / 2) + 1 + 3 = 0 + 1 + 3 = 4 elements. (Notice 1/2 is 0, not 0.5 or 1)
  • i = 2: (2 / 2) + 2 + 3 = 1 + 2 + 3 = 6 elements.
  • Total: 3 + 4 + 6 = 13 elements.
Nested Loops

2. The Triple Matrix Traversal (2016-17)

How many times exactly will "hello" be printed?

int main(void) { 
    int i, j, k; 
    int count = 0;
    
    for(i=1; i<=4; i++) 
        for(j=1; j<=i; j++) 
            for(k=1; k<=j; k++) 
                count++;
                
    printf("%d", count); 
    return 0; 
}

Answer: 20

The Algorithm: You have to compute the "Sum of the Sums". The innermost loop runs `j` times.

  • When i = 1: `j` goes up to 1. Innermost loop runs `1` time. (Sum = 1)
  • When i = 2: `j` goes 1, then 2. Innermost loop runs `1 + 2 = 3` times.
  • When i = 3: `j` goes 1, 2, 3. Innermost loop runs `1 + 2 + 3 = 6` times.
  • When i = 4: `j` goes 1, 2, 3, 4. Innermost loop runs `1 + 2 + 3 + 4 = 10` times.

Total: 1 + 3 + 6 + 10 = 20.

II. Memory Mutilation

Pointers

3. Pointer Type Ignorance (2016-17)

Assuming short int is 2 bytes and float is 4 bytes. What is the difference between the printed addresses?

void main() { 
    float b = 10.0; 
    short int *p = (short int*)&b; 
    
    printf("%u\t", p); 
    p++; 
    printf("%u", p); 
}

Answer: 2

The Trick: Pointer arithmetic depends entirely on the declared type of the pointer, not the type of the variable it actually points to in memory.

Even though `p` points to a `float` (which takes 4 bytes), `p` is declared as a short int*. When you do p++, the compiler moves the address forward by sizeof(short int).
The difference printed will be exactly 2 bytes.

Dangerous

4. The Use-After-Free Illusion (2024-25)

#include <stdio.h>
#include <stdlib.h>

void func() {
    int *p = malloc(sizeof(int));
    *p = 42;
    printf("Addr: %p\n", p);
    free(p);
    
    *p = 24;  // <-- CRITICAL LINE
    printf("Addr: %p\n", p);
}

int main() {
    func();
    return 0;
}

Answer: Executes without immediate errors, but behavior is Undefined.

The Concept: This asks what physically happens inside the computer.

When you call free(p), the memory is handed back to the OS heap manager, but the pointer `p` is NOT automatically set to NULL. It still holds the exact same memory address. This is called a "Dangling Pointer".

When you subsequently assign *p = 24, the C compiler happily writes the number 24 to that memory address. It likely will not crash immediately (unless the OS already reclaimed that page), making it one of the most dangerous, silent bugs in C programming: Undefined Behavior via Use-After-Free.

III. The Base-N Crucible

Base Math

5. The Octal-Hex Collision (2014-15)

Assuming 16-bit 2's complement representation.
If X = (EA)₁₆ and Y = (71)₈. Perform the operation: -X - Y.
The answer in hexadecimal system is:

Answer: (FEDD)₁₆

The Math: Convert everything to decimal first, perform the math, then convert back to 16-bit 2's complement hex.

  • X = EA (hex): (14 * 16) + (10 * 1) = 224 + 10 = 234
  • Y = 71 (octal): (7 * 8) + (1 * 1) = 56 + 1 = 57
  • Operation: -X - Y = -234 - 57 = -291

Converting -291 to 16-bit Hex:

  1. Positive 291 in Hex: 0123 (since 256 + 32 + 3)
  2. Invert bits (1's complement): FEDC
  3. Add 1 (2's complement): FEDD
Nesting

6. The Assignment Inception (2024-25)

#include <stdio.h>

int func1(int a) {
    static int b = 5;
    if (b < 9) {
        return func1(++a/2 + b++) + (++a/2 + b++);
    } else {
        return (++a/2 + b++);
    }
}

// Ignore func2, assume func2 is identical to func1 for this exercise.
// The actual exam asks what happens when evaluating nested arguments:
int main() {
    int res1 = 0, res2 = 0;
    
    // Evaluate this statement:
    printf("%d\n", func1(res1 = func1(res2 = func1(20))));
    return 0;
}

Answer: Evaluates from deepest parentheses outward, mutating static state globally.

The Trick: You are not just passing values; you are executing embedded assignment operators within the function arguments.

In C, before the outermost func1(...) can execute, its arguments must be resolved.
1. It executes the innermost: func1(20). Because of `static int b`, this mutates the global state of `b`.
2. The return value is assigned to `res2`. This assignment expression resolves to the value of `res2`.
3. It executes the middle: func1(res2). `b` continues mutating.
4. The return value is assigned to `res1`.
5. Finally, the outermost function func1(res1) executes.

This is an endurance test designed to make students lose track of the static `b` variable over 3 sequential evaluations.
